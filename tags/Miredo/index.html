在Debian Strech上开启Miredo

Miredo是Debian/Ubuntu上的类Teredo服务，通常可以用它来访问一些ipv6的网络，除此之外，每台Miredo的客户机都可以分到一个2001开头的ipv6 。 这个ipv6对于内网的BT软件是否有实质性的帮助一直都没有确切的说法，但使用Miredo一段时间后观察数据流量，可以发现确实从它上面走了一些流量，可见它虽然帮助不大，但聊胜于无。 Debian的官方源自带Miredo，所以 [cc]apt install miredo[/cc] 查看网卡信息 [cc]ifconfig[/cc] 出现名为teredo的接口即表示安装成功 若出现RTNETLINK permission denied的错误提示 则需要在sysctl.conf的末尾下添加以下几项 [cc] vim /etc/sysctl.conf net.ipv6.conf.all.disable_ipv6 = 0 net.ipv6.conf.default.disable_ipv6 = 0 net.ipv6.conf.lo.disable_ipv6 = 0 [/cc] 保存退出重启，这时会遇到另外一个问题。Miredo的默认启动优先级要高于域名解析服务，于是它在每次开机的时候都会因为无法正常解析中继服务器的域名而导致启动失败。 最简单的办法是将这个地址改成IP [cc]vim /etc/miredo.conf[/cc] 将原有的地址注释掉，添加一条 [cc]ServerAddress 83.170.6.76[/cc] 保存重启后，检查Miredo运行状况 [cc]service miredo status[/cc] 一切正常 尝试ping以下ipv6的域名 依旧正常 这样就算是安装配置完毕了。
